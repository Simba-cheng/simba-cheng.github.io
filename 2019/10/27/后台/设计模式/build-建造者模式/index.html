    之前写 elasticsearch java client，在构造 HTTP客户端 时，发现用了builder建造者模式，而且在《Effective Java》第一章提到过，当时没细看，于是记在心头。

    现在空闲了，来瞅瞅看

    builder 建造者模式

    目的


    分离 对象的属性 与 对象创建的过程

    敲黑板，划重点！

    作用

    builder建造者模式 是 Java对象 构造方法 的代替方案。

    试想一个对象有几十个属性(成员变量)，我们需要使用它时，需要了解每个属性的含义，为每一个属性set值，是不是非常麻烦？

    builder建造者模式 可以解决以上痛点：

    对象中属性众多，重要的就那么几个，因此，建造者模式中，开发者需要指定一些重要的属性，然后让建造者去实现构建对象的过程。

这就是对象的属性与创建分离

这样 对于建造者而言 隐藏了复杂的对象构建细节

    Demo-1

    ReactorConfig对象 有两个构造方法，一个是默认形式，一个是入参形式。

    两个构造函数都是 private修饰符，这样外部就无法通过new方法来构造 ReactorConfig对象，(当然也可以放开)

    ReactorConfig对象 的创建托管给了 常量DEFAULT 和 Builder静态内部类

    package org.code.builder;

/**
 * @author: CYX
 * @create: 2019-10-10 20:23
 **/
public class AppCode {
    public static void main(String[] args) {
        ReactorConfigBuilder reactorConfig = ReactorConfigBuilder.custom().setIoThreadCount(66).build();
        System.out.println(reactorConfig.toString());

        ReactorCacheBuilder cacheBuilder = ReactorCacheBuilder.newBuilder().initialCapacity(1).maximumSize(2).maximumWeight(3);
        System.out.println(cacheBuilder.toString());
    }
}


/**
 * 建造者模式
 * <p>
 * 参考IOReactorConfig
 *
 * @author: CYX
 * @create: 2019-10-10 19:34
 * @see org.apache.http.impl.nio.reactor.IOReactorConfig
 **/
public final class ReactorConfig {

    /**
     * 获取可用于Java虚拟机的处理器数量
     */
    private static final int AVAIL_PROCS = Runtime.getRuntime().availableProcessors();

    public static final ReactorConfig DEFAULT = new Builder().build();

    private long selectInterval;
    private long shutdownGracePeriod;
    private boolean interestOpQueued;
    private int ioThreadCount;
    private int soTimeout;
    private boolean soReuseAddress;
    private int soLinger;
    private boolean soKeepAlive;
    private boolean tcpNoDelay;
    private int connectTimeout;
    private int sndBufSize;
    private int rcvBufSize;
    private int backlogSize;

    private ReactorConfig() {
    }

    private ReactorConfig(
            final long selectInterval,
            final long shutdownGracePeriod,
            final boolean interestOpQueued,
            final int ioThreadCount,
            final int soTimeout,
            final boolean soReuseAddress,
            final int soLinger,
            final boolean soKeepAlive,
            final boolean tcpNoDelay,
            final int connectTimeout,
            final int sndBufSize,
            final int rcvBufSize,
            final int backlogSize) {
        this.selectInterval = selectInterval;
        this.shutdownGracePeriod = shutdownGracePeriod;
        this.interestOpQueued = interestOpQueued;
        this.ioThreadCount = ioThreadCount;
        this.soTimeout = soTimeout;
        this.soReuseAddress = soReuseAddress;
        this.soLinger = soLinger;
        this.soKeepAlive = soKeepAlive;
        this.tcpNoDelay = tcpNoDelay;
        this.connectTimeout = connectTimeout;
        this.sndBufSize = sndBufSize;
        this.rcvBufSize = rcvBufSize;
        this.backlogSize = backlogSize;
    }

    public long getSelectInterval() {
        return selectInterval;
    }

    public long getShutdownGracePeriod() {
        return shutdownGracePeriod;
    }

    public boolean isInterestOpQueued() {
        return interestOpQueued;
    }

    public int getIoThreadCount() {
        return ioThreadCount;
    }

    public int getSoTimeout() {
        return soTimeout;
    }

    public boolean isSoReuseAddress() {
        return soReuseAddress;
    }

    public int getSoLinger() {
        return soLinger;
    }

    public boolean isSoKeepAlive() {
        return soKeepAlive;
    }

    public boolean isTcpNoDelay() {
        return tcpNoDelay;
    }

    public int getConnectTimeout() {
        return connectTimeout;
    }

    public int getSndBufSize() {
        return sndBufSize;
    }

    public int getRcvBufSize() {
        return rcvBufSize;
    }

    public int getBacklogSize() {
        return backlogSize;
    }

    public static Builder custom() {
        return new Builder();
    }

    public static class Builder {

        private long selectInterval;
        private long shutdownGracePeriod;
        private boolean interestOpQueued;
        private int ioThreadCount;
        private int soTimeout;
        private boolean soReuseAddress;
        private int soLinger;
        private boolean soKeepAlive;
        private boolean tcpNoDelay;
        private int connectTimeout;
        private int sndBufSize;
        private int rcvBufSize;
        private int backlogSize;

        //
        public Builder() {
            this.selectInterval = 1000;
            this.shutdownGracePeriod = 500;
            this.interestOpQueued = false;
            this.ioThreadCount = AVAIL_PROCS;
            this.soTimeout = 0;
            this.soReuseAddress = false;
            this.soLinger = -1;
            this.soKeepAlive = false;
            this.tcpNoDelay = true;
            this.connectTimeout = 0;
            this.sndBufSize = 0;
            this.rcvBufSize = 0;
            this.backlogSize = 0;
        }

        public Builder setSelectInterval(long selectInterval) {
            this.selectInterval = selectInterval;
            return this;
        }

        public Builder setShutdownGracePeriod(long shutdownGracePeriod) {
            this.shutdownGracePeriod = shutdownGracePeriod;
            return this;
        }

        public Builder setInterestOpQueued(boolean interestOpQueued) {
            this.interestOpQueued = interestOpQueued;
            return this;
        }

        public Builder setIoThreadCount(int ioThreadCount) {
            this.ioThreadCount = ioThreadCount;
            return this;
        }

        public Builder setSoTimeout(int soTimeout) {
            this.soTimeout = soTimeout;
            return this;
        }

        public Builder setSoReuseAddress(boolean soReuseAddress) {
            this.soReuseAddress = soReuseAddress;
            return this;
        }

        public Builder setSoLinger(int soLinger) {
            this.soLinger = soLinger;
            return this;
        }

        public Builder setSoKeepAlive(boolean soKeepAlive) {
            this.soKeepAlive = soKeepAlive;
            return this;
        }

        public Builder setTcpNoDelay(boolean tcpNoDelay) {
            this.tcpNoDelay = tcpNoDelay;
            return this;
        }

        public Builder setConnectTimeout(int connectTimeout) {
            this.connectTimeout = connectTimeout;
            return this;
        }

        public Builder setSndBufSize(int sndBufSize) {
            this.sndBufSize = sndBufSize;
            return this;
        }

        public Builder setRcvBufSize(int rcvBufSize) {
            this.rcvBufSize = rcvBufSize;
            return this;
        }

        public Builder setBacklogSize(int backlogSize) {
            this.backlogSize = backlogSize;
            return this;
        }

        public ReactorConfig build() {
            return new ReactorConfig(
                    selectInterval, shutdownGracePeriod, interestOpQueued, ioThreadCount,
                    soTimeout, soReuseAddress, soLinger, soKeepAlive, tcpNoDelay,
                    connectTimeout, sndBufSize, rcvBufSize, backlogSize);
        }
    }

    @Override
    public String toString() {
        final StringBuilder builder = new StringBuilder();
        builder.append("[selectInterval=").append(this.selectInterval)
                .append(", shutdownGracePeriod=").append(this.shutdownGracePeriod)
                .append(", interestOpQueued=").append(this.interestOpQueued)
                .append(", ioThreadCount=").append(this.ioThreadCount)
                .append(", soTimeout=").append(this.soTimeout)
                .append(", soReuseAddress=").append(this.soReuseAddress)
                .append(", soLinger=").append(this.soLinger)
                .append(", soKeepAlive=").append(this.soKeepAlive)
                .append(", tcpNoDelay=").append(this.tcpNoDelay)
                .append(", connectTimeout=").append(this.connectTimeout)
                .append(", sndBufSize=").append(this.sndBufSize)
                .append(", rcvBufSize=").append(this.rcvBufSize)
                .append(", backlogSize=").append(this.backlogSize)
                .append("]");
        return builder.toString();
    }
}

    ReactorCache 对象 的创建托管给了 Builder静态方法，然后

    package org.code.builder;

/**
 * 建造者模式
 * 
 * 参考：com.google.common.cache.CacheBuilder
 *
 * @author: CYX
 * @create: 2019-10-11 19:26
 **/
public final class ReactorCache {

    private static final int UNSET_INT = -1;

    /**
     * 设置内部哈希表的最小总大小
     */
    private int initialCapacity = UNSET_INT;

    /**
     * 最大数量
     */
    private long maximumSize = UNSET_INT;

    /**
     * 最大权重
     */
    private long maximumWeight = UNSET_INT;

    private ReactorCache() {
    }

    public static ReactorCache newBuilder() {
        return new ReactorCache();
    }

    /**
     * 设置内部哈希表的最小总大小
     *
     * @return
     */
    public ReactorCache initialCapacity(int initialCapacity) {
        //check
        this.initialCapacity = initialCapacity;
        return this;
    }

    /**
     * 最大数量
     *
     * @param maximumSize
     * @return
     */
    public ReactorCache maximumSize(long maximumSize) {
        //check
        this.maximumSize = maximumSize;
        return this;
    }

    /**
     * 最大权重
     *
     * @param maximumWeight
     * @return
     */
    public ReactorCache maximumWeight(long maximumWeight) {
        this.maximumWeight = maximumWeight;
        return this;
    }

    @Override
    public String toString() {
        final StringBuilder sb = new StringBuilder("ReactorCache{");
        sb.append("initialCapacity=").append(initialCapacity);
        sb.append(", maximumSize=").append(maximumSize);
        sb.append(", maximumWeight=").append(maximumWeight);
        sb.append('}');
        return sb.toString();
    }
}

    实现

    org.apache.http.impl.nio.reactor.IOReactorConfig、com.google.common.cache.CacheBuilder

    翻阅了一些源码（参照以上两个类），建造者模式有两种实现写法：

    • 直接new

      在Builder里面直接new对象的方式，如 ReactorCache、com.google.common.cache.CacheBuilder 中的实现方法。

    • 间接new

      间接new对象方式，如 ReactorConfig、org.apache.http.impl.nio.reactor.IOReactorConfig 中的实现方法。
