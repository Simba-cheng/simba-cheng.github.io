    pageCache е’Ң zeroCopyпјҢиҝҷдёӨдёӘзҹҘиҜҶзӮ№йңҖиҰҒдёҚе°‘LinuxзҹҘиҜҶиғҢжҷҜпјӣ

    жҲ‘жҠҠзӣёе…іиғҢжҷҜзҹҘиҜҶж•ҙзҗҶеҮәжқҘпјҢеҰӮжһңжңүз–‘й—®пјҢеҸҜд»ҘеңЁжҹҘйҳ…вҖқеҸӮиҖғиө„ж–ҷвҖқдёӢиҜҰз»Ҷй“ҫжҺҘпјҢжҲ–дёҺжҲ‘иҒ”зі»дёҖиө·и®Ёи®әгҖӮгҖӮ

    Kafka дёәд»Җд№Ҳиҝҷд№Ҳеҝ«пјҹ

    KafkaеңЁеә•еұӮж‘’ејғдәҶJavaе Ҷзј“еӯҳжңәеҲ¶пјҢйҮҮз”ЁдәҶж“ҚдҪңзі»з»ҹзә§еҲ«зҡ„ йЎөзј“еӯҳпјҲPage CacheпјүпјҢеҗҢж—¶е°ҶйҡҸжңәеҶҷж“ҚдҪңж”№дёә йЎәеәҸеҶҷпјҢеҶҚз»“еҗҲ Zero-Copy зҡ„зү№жҖ§жһҒеӨ§ең°ж”№е–„дәҶIOжҖ§иғҪгҖӮ

    еҗҢж—¶KafkaиҝҳдҪҝз”ЁдәҶ еҲҶеҢә(partition)пјҢйҖҡиҝҮе°Ҷtopicзҡ„ж¶ҲжҒҜжү“ж•ЈеҲ°еӨҡдёӘеҲҶеҢәе№¶еҲҶеёғдҝқеӯҳеңЁдёҚеҗҢзҡ„brokerдёҠе®һзҺ°дәҶж¶ҲжҒҜеӨ„зҗҶ(дёҚз®ЎжҳҜproducerиҝҳжҳҜconsumer)зҡ„й«ҳеҗһеҗҗйҮҸгҖӮ

    (е’Ң elasticsearch еҲҶзүҮе·®дёҚеӨҡзҡ„жҖқи·Ҝ)

    Kafkaеҝ«йҷӨдәҶдёҠйқўжҸҗеҲ°зҡ„еӣӣзӮ№еӨ–пјҢиҝҳеңЁе…¶д»–ең°ж–№еҒҡдәҶдјҳеҢ–пјҢжңүе…ҙи¶Јзҡ„еҸҜд»Ҙйҳ…иҜ»дёҖдёӢиҝҷзҜҮеҚҡж–ҮпјҡгҖҠWhy Kafka Is so FastгҖӢ

    з”ЁжҲ·жҖҒгҖҒеҶ…ж ёжҖҒ

    ж“ҚдҪңзі»з»ҹзҡ„ж ёеҝғжҳҜеҶ…ж ё(kernel)пјҢе®ғзӢ¬з«ӢдәҺжҷ®йҖҡзҡ„еә”з”ЁзЁӢеәҸпјҢеҸҜд»Ҙи®ҝй—®еҸ—дҝқжҠӨзҡ„еҶ…еӯҳз©әй—ҙпјҢд№ҹжңүи®ҝй—®еә•еұӮзЎ¬д»¶и®ҫеӨҮзҡ„жүҖжңүжқғйҷҗ(CPUиө„жәҗгҖҒеӯҳеӮЁиө„жәҗгҖҒI/Oиө„жәҗзӯү)гҖӮ

    д»Һе®Ҹи§ӮдёҠжқҘзңӢпјҢLinuxж“ҚдҪңзі»з»ҹзҡ„дҪ“зі»жһ¶жһ„еҲҶдёә з”ЁжҲ·жҖҒ е’Ң еҶ…ж ёжҖҒгҖӮ

    еҶ…ж ёжҖҒпјҡжҺ§еҲ¶и®Ўз®—жңәзҡ„зЎ¬д»¶иө„жәҗпјҢе№¶жҸҗдҫӣдёҠеұӮеә”з”ЁзЁӢеәҸиҝҗиЎҢзҡ„зҺҜеўғгҖӮжҜ”еҰӮ socket I/Oж“ҚдҪңгҖҒж–Үд»¶зҡ„иҜ»еҶҷзӯүж“ҚдҪңгҖӮ

    з”ЁжҲ·жҖҒпјҡдёҠеұӮеә”з”ЁзЁӢеәҸзҡ„жҙ»еҠЁз©әй—ҙпјҢеә”з”ЁзЁӢеәҸзҡ„жү§иЎҢеҝ…йЎ»дҫқжүҳдәҺеҶ…ж ёжҸҗдҫӣзҡ„иө„жәҗпјҢеҢ…жӢ¬CPUиө„жәҗгҖҒеӯҳеӮЁиө„жәҗгҖҒI/Oиө„жәҗзӯүгҖӮ

    зі»з»ҹи°ғз”ЁпјҡдёәдәҶдҝқиҜҒеҶ…ж ёзҡ„е®үе…ЁпјҢзҺ°еңЁзҡ„ж“ҚдҪңзі»з»ҹдёҖиҲ¬йғҪејәеҲ¶з”ЁжҲ·иҝӣзЁӢдёҚиғҪзӣҙжҺҘж“ҚдҪңеҶ…ж ёгҖӮдёәдәҶдҪҝдёҠеұӮеә”з”ЁиғҪеӨҹи®ҝй—®еҲ°иҝҷдәӣиө„жәҗпјҢеҶ…ж ёеҝ…йЎ»дёәдёҠеұӮеә”з”ЁжҸҗдҫӣи®ҝй—®зҡ„жҺҘеҸЈпјҡеҚізі»з»ҹи°ғз”ЁгҖӮ

    з”ЁжҲ·з©әй—ҙгҖҒеҶ…ж ёз©әй—ҙ

    еҜ№32дҪҚж“ҚдҪңзі»з»ҹиҖҢиЁҖпјҢе®ғзҡ„иҷҡжӢҹз©әй—ҙдёә4GпјҲ2зҡ„32ж¬Ўж–№пјүпјҢд№ҹе°ұжҳҜиҜҙдёҖдёӘиҝӣзЁӢзҡ„жңҖеӨ§ең°еқҖз©әй—ҙдёә4GгҖӮ

    ж“ҚдҪңзі»з»ҹе°ҶиҷҡжӢҹең°еқҖз©әй—ҙеҲ’еҲҶдёәдёӨйғЁеҲҶпјҢдёҖйғЁеҲҶдёә еҶ…ж ёз©әй—ҙпјҢеҸҰдёҖйғЁеҲҶдёә з”ЁжҲ·з©әй—ҙгҖӮ

    й’ҲеҜ№Linuxж“ҚдҪңзі»з»ҹиҖҢиЁҖпјҢжңҖй«ҳзҡ„1Gеӯ—иҠӮ(д»ҺиҷҡжӢҹең°еқҖ 0xC0000000 еҲ° 0xFFFFFFFF)з”ұеҶ…ж ёдҪҝз”ЁпјҢз§°дёәеҶ…ж ёз©әй—ҙгҖӮ
иҖҢиҫғдҪҺзҡ„3Gеӯ—иҠӮ(д»ҺиҷҡжӢҹең°еқҖ 0x00000000 еҲ° 0xBFFFFFFF)з”ұеҗ„дёӘиҝӣзЁӢдҪҝз”ЁпјҢз§°дёәз”ЁжҲ·з©әй—ҙгҖӮ

    еҶ…ж ёз©әй—ҙжҳҜеҶ…ж ёд»Јз ҒиҝҗиЎҢзҡ„ең°ж–№пјҢз”ЁжҲ·з©әй—ҙжҳҜз”ЁжҲ·зЁӢеәҸд»Јз ҒиҝҗиЎҢзҡ„ең°ж–№

    з”ЁжҲ·жҖҒе’ҢеҶ…ж ёжҖҒзҡ„еҲҮжҚў

    еӣ дёәж“ҚдҪңзі»з»ҹзҡ„иө„жәҗжҳҜжңүйҷҗзҡ„пјҢдёәдәҶеҮҸе°‘жңүйҷҗиө„жәҗзҡ„и®ҝй—®е’ҢдҪҝз”ЁеҶІзӘҒпјҢUnix/Linuxзҡ„и®ҫи®Ўе“ІеӯҰд№ӢдёҖе°ұжҳҜпјҡеҜ№дёҚеҗҢзҡ„ж“ҚдҪңиөӢдәҲдёҚеҗҢзҡ„жү§иЎҢзӯүзә§пјҢе°ұжҳҜжүҖи°“зү№жқғзҡ„жҰӮеҝөгҖӮ

    з®ҖеҚ•иҜҙе°ұжҳҜжңүеӨҡеӨ§иғҪеҠӣеҒҡеӨҡеӨ§зҡ„дәӢпјҢдёҺзі»з»ҹзӣёе…ізҡ„дёҖдәӣзү№еҲ«е…ій”®зҡ„ж“ҚдҪңеҝ…йЎ»з”ұжңҖй«ҳзү№жқғзҡ„зЁӢеәҸжқҘе®ҢжҲҗгҖӮ

    Intelзҡ„X86жһ¶жһ„зҡ„CPUжҸҗдҫӣдәҶ0еҲ°3еӣӣдёӘзү№жқғзә§пјҢж•°еӯ—и¶Ҡе°ҸпјҢзү№жқғи¶Ҡй«ҳпјҢLinuxж“ҚдҪңзі»з»ҹдёӯдё»иҰҒйҮҮз”ЁдәҶ0е’Ң3дёӨдёӘзү№жқғзә§пјҢеҲҶеҲ«еҜ№еә”зҡ„е°ұжҳҜеҶ…ж ёжҖҒе’Ңз”ЁжҲ·жҖҒгҖӮ

    иҝҗиЎҢдәҺз”ЁжҲ·жҖҒзҡ„иҝӣзЁӢеҸҜд»Ҙжү§иЎҢзҡ„ж“ҚдҪңе’Ңи®ҝй—®зҡ„иө„жәҗйғҪдјҡеҸ—еҲ°жһҒеӨ§зҡ„йҷҗеҲ¶пјҢиҖҢиҝҗиЎҢеңЁеҶ…ж ёжҖҒзҡ„иҝӣзЁӢеҲҷеҸҜд»Ҙжү§иЎҢд»»дҪ•ж“ҚдҪңе№¶дё”еңЁиө„жәҗзҡ„дҪҝз”ЁдёҠжІЎжңүйҷҗеҲ¶гҖӮ

    дҫӢеҰӮпјҡJVMеұһдәҺз”ЁжҲ·жҖҒпјҢж— жқғи®ҝй—®еҶ…ж ёпјҢеҸӘиғҪйҖҡиҝҮзі»з»ҹи°ғз”Ёе®ҢжҲҗз”ЁжҲ·жҖҒеҲ°еҶ…ж ёжҖҒзҡ„и°ғз”ЁпјҲJNIи°ғз”Ёпјү

    еҫҲеӨҡзЁӢеәҸејҖе§Ӣж—¶иҝҗиЎҢдәҺз”ЁжҲ·жҖҒпјҢдҪҶеңЁжү§иЎҢзҡ„иҝҮзЁӢдёӯпјҢдёҖдәӣж“ҚдҪңйңҖиҰҒеңЁеҶ…ж ёжқғйҷҗдёӢжүҚиғҪжү§иЎҢпјҢиҝҷе°ұж¶үеҸҠеҲ°д»Һ з”ЁжҲ·жҖҒдёҺеҶ…ж ёжҖҒд№Ӣй—ҙзҡ„еҲҮжҚўгҖӮ

    з”ЁжҲ·жҖҒ е’Ң еҶ…ж ёжҖҒ еҲҮжҚўиҖ—ж—¶гҖҒд»Јд»·еҫҲеӨ§гҖӮ

    Furthermore, we are building on top of the JVM, and anyone who has spent any time with Java memory usage knows two things:

    1.The memory overhead of objects is very high, often doubling the size of the data stored (or worse).

    2.Java garbage collection becomes increasingly fiddly and slow as the in-heap data increases.

    Furthermore, this cache will stay warm even if the service is restarted, whereas the in-process cache will need to be rebuilt in memory (which for a 10GB cache may take 10 minutes) or else it will need to start with a completely cold cache (which likely means terrible initial performance). This also greatly simplifies the code as all logic for maintaining coherency between the cache and filesystem is now in the OS, which tends to do so more efficiently and more correctly than one-off in-process attempts. If your disk usage favors linear reads then read-ahead is effectively pre-populating this cache with useful data on each disk read.
