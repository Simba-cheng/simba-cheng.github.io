    代理模式

    模式动机：

    在某些情况下，一个客户不想或者不能直接引用一个对象，此时可以通过一个称之为”代理”的第三者来实现间接引用。

    代理对象可以在客户端和目标对象之间起到中介的作用，并且可以通过代理对象去掉客户不能看到的内容和服务或者添加客户需要的额外服务。


    模式定义：

    给某一个对象提供一个代理，并由代理对象控制对原对象的引用。

    静态代理

    先看代码示例

    静态代理-示例代码-1

    /**
 * 接口 - 抽象角色
 *
 * @author cyx
 * @date 2020/5/14 16:03
 */
public interface Subject {
    void save();
}

/**
 * 实现类 - 真正角色
 *
 * @author cyx
 * @date 2020/5/14 16:14
 */
public class RealSubject implements Subject {
    @Override
    public void save() {
        System.out.println("保存数据...");
    }
}

/**
 * 代理类 - 代理角色
 *
 * @author cyx
 * @date 2020/5/14 16:15
 */
public class SubjectProxy implements Subject {

    private Subject subject;

    public SubjectProxy(Subject subject) {
        this.subject = subject;
    }

    @Override
    public void save() {
        System.out.println("111111111");
        //执行目标方法
        subject.save();
        System.out.println("222222222");
    }
}

/**
 * @author cyx
 * @date 2020/5/14 16:17
 */
public class MainApp {

    public static void main(String[] args) {
        //真正角色
        RealSubject oneUserService = new RealSubject();

        //代理
        SubjectProxy userServiceProxy = new SubjectProxy(oneUserService);

        //执行代理
        userServiceProxy.save();
    }

}

    输出结果：

    111111111
保存数据...
222222222

 * 抽象角色

public abstract class Subject {

    public abstract void request();

/**
 * 实现类 - 真正角色
 *
 * @author cyx
 * @date 2020/5/15 11:14
  */
public class RealSubject extends Subject {

    @Override
    public void request() {
        System.out.println("........process.......");
    }
}

/**
 * @author cyx
 * @date 2020/5/15 11:16
  */
public class SubjectProxy extends Subject {

    private Subject subject;

    public SubjectProxy() {
        this.subject = new RealSubject();
    }

    public SubjectProxy(Subject subject) {
        this.subject = subject;
    }

    @Override
    public void request() {

        System.out.println("前置处理...");

        subject.request();

        System.out.println("后置处理...");
    }
}

/**
 * @author cyx

public class MainAppTest {

    public static void main(String[] args) {

        RealSubject realSubject = new RealSubject();

        //SubjectProxy proxy = new SubjectProxy(realSubject);
        SubjectProxy proxy = new SubjectProxy();

        proxy.request();
    }
}

    输出结果：

    前置处理...
........process.......
后置处理...

    可以看到，代理模式可以在不修改被代理的基础上，通过代理类的扩展，进行一些功能增强。

    不过需要注意，代理类 和 被代理类应该共同实现一个接口，或者共同继承某个类。

    上面是静态代理的示例，为什么叫静态？因为代理的类型都是之前事先定义好的。

    动态代理

    先看下代码示例

    动态代理示例

    /**
 * 接口 - 抽象角色
 *
 * @author cyx
 * @date 2020/5/15 11:39
 */
public interface SellWine {

    void process();

}

/**
 * 茅台 - 柜台
 * <p>
 * 实现类 - 真正角色
 *
 * @author cyx
 * @date 2020/5/15 11:40
 */
public class MaotaiCounter implements SellWine {

    @Override
    public void process() {
        System.out.println("茅台酒...");
    }
}

/**
 * 万达 - 代理商
 *
 * @author cyx
 * @date 2020/5/15 14:17
 */
public class WanDaProxy implements InvocationHandler {

    private Object object;

    public WanDaProxy(Object object) {
        this.object = object;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("卖酒前置处理...");

        method.invoke(object, args);

        System.out.println("卖酒后置处理...");
        return null;
    }
}

/**
 * @author cyx
 * @date 2020/5/15 14:25
 */
public class MainAppSellWine {

    public static void main(String[] args) {

        //实例化
        MaotaiCounter maotaiCounter = new MaotaiCounter();

        InvocationHandler invocationHandler = new WanDaProxy(maotaiCounter);

        SellWine dynamicProxy = (SellWine) Proxy.newProxyInstance(MaotaiCounter.class.getClassLoader(),
                MaotaiCounter.class.getInterfaces(), invocationHandler);

        dynamicProxy.process();
    }

}

    输出结果：

    卖酒前置处理...
茅台酒...
卖酒后置处理...

    在上面 WanDaProxy 代理类中，并没有事先指定某个类、接口。这就是与上面静态代理最大的区别。

    上面代码中，有个重要的 Proxy 类，通过 Proxy.newProxyInstance() 方法才会动态创建代理。

    Object newProxyInstance(ClassLoader loader,Class<?>[] interfaces,InvocationHandler h)

    loader：类加载器

    interfaces：被代理接口

    h：InvocationHandler 接口实现

    InvocationHandler

    InvocationHandler接口 是 proxy代理实例 调用处理程序的一个接口；每一个proxy代理实例都有一个关联的调用处理程序；

    每一个动态代理类的调用程序都必须实现 InvocationHandler 接口。当我们通过动态代理对象调用一个方法时，这个方法的调用就会被转发到实现 InvocationHandler 接口类的 invoke() 方法调用。

    InvocationHandler 内部只是一个 invoke() 方法

    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable;

    proxy：代理对象

    method：代理对象调用的方法

    args：调用的方法中的参数

    proxy 动态产生的代理会调用 InvocationHandler 实现类，所以 InvocationHandler 是实际执行者。


    动态代理利用了 JDK API，又被称为”JDK代理、接口代理”，动态的在内存中构建代理对象，从而实现对目标对象的动态代理功能。

    静态代理 和 动态代理 区别

    1. 静态代理在编译时已经实现，编译完成后代理类便是一个实际的class文件。
    2. 动态代理在运行时动态生成，即编译后没有class文件，而是在运行时动态生成，并加载到JVM中。
