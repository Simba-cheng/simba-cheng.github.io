    DataStream Transformations 转换

    Map

    DataStream → DataStream

    常用来对 数据/元素 转换，输入和输出为一对一关系

    示例:

    private void testMap() throws Exception {

  StreamExecutionEnvironment environment = StreamExecutionEnvironment.getExecutionEnvironment();
  environment.setParallelism(1);

  String message = "{\"adtype\":\"TUIJIAN\",\"blid\":\"1335557523\",\"promotionDate\":\"20200320\",\"reqClt\":\"app\",\"segClick\":\"1\",\"segCost\":\"1408872460\",\"sid\":\"0\"}\n";

  SingleOutputStreamOperator<ClickData> dataStream = environment.fromElements(message)
          .map(new MapFunction<String, ClickData>() {
              @Override
              public ClickData map(String value) throws Exception {
                  return JSONObject.toJavaObject(JSONObject.parseObject(value), ClickData.class);
              }
          });
  dataStream.print("map");
  environment.execute("test map");
}

    FlatMap

    DataStream → DataStream

    常用来对 数据/元素 转换，输入和输出为1对n的关系，n大于等于0，此方法可以将一行数据拆成多行

    示例：

    private void testFlatMap() throws Exception {
    StreamExecutionEnvironment environment = StreamExecutionEnvironment.getExecutionEnvironment();
    environment.setParallelism(1);

    SingleOutputStreamOperator<String> dataStream = environment.fromElements("1,2,4,5,2,34,23,4,234,324")
            .flatMap(new FlatMapFunction<String, String>() {
                @Override
                public void flatMap(String value, Collector<String> out) throws Exception {
                    for (String word : value.split(",")) {
                        out.collect(word.trim());
                    }
                }
            });
    dataStream.print("flatMap");
    environment.execute("test flatMap");
}

    Filter

    DataStream → DataStream

    常用来对 数据 过滤/筛选

    示例：

    private void testFilter() throws Exception {

    StreamExecutionEnvironment environment = StreamExecutionEnvironment.getExecutionEnvironment();
    environment.setParallelism(1);

    SingleOutputStreamOperator<String> dataStream = environment.fromElements("009", "4", "8", "2", "53", "43", "34", "234")
            .filter(new FilterFunction<String>() {
                @Override
                public boolean filter(String value) throws Exception {
                    return value.length() == 2;
                }
            });
    dataStream.print("filter");
    environment.execute("test filter");
}

    KeyBy

    DataStream → KeyedStream

    常用来 对数据切分，把相同key的数据放到同一个逻辑分区中

    示例：

    private void testKeyBy() throws Exception {
    StreamExecutionEnvironment environment = StreamExecutionEnvironment.getExecutionEnvironment();
    environment.setParallelism(2);
    DataStreamSource<Tuple2<String, Integer>> input = environment
            .fromElements(new Tuple2<>("foo", 1), new Tuple2<>("foo", 2),
                    new Tuple2<>("bar", 3), new Tuple2<>("baz", 4),
                    new Tuple2<>("bar", 4), new Tuple2<>("baz", 5));
    input.print("input data");
    KeyedStream<Tuple2<String, Integer>, Tuple> keyed = input.keyBy(0);
    keyed.print("keyBy data");

    environment.execute("test keyBy");
}

    Reduce

    KeyedStream → DataStream

    基于KeyedStream(keyBy)，对相同的key的数据进行reduce操作

    示例：

    private void testReduce() throws Exception {

    StreamExecutionEnvironment environment = StreamExecutionEnvironment.getExecutionEnvironment();
    environment.setParallelism(2);

    DataStream<Tuple2<String, Integer>> input = environment.fromElements(
            new Tuple2<>("foo", 1), new Tuple2<>("foo", 2),
            new Tuple2<>("bar", 3), new Tuple2<>("baz", 4),
            new Tuple2<>("bar", 4), new Tuple2<>("baz", 5));

    KeyedStream<Tuple2<String, Integer>, Tuple> keyed = input.keyBy(0);

    DataStream<Tuple2<String, Integer>> out = keyed.reduce(new ReduceFunction<Tuple2<String, Integer>>() {
        @Override
        public Tuple2<String, Integer> reduce(Tuple2<String, Integer> value1, Tuple2<String, Integer> value2) throws Exception {
            return new Tuple2<>(value1.f0, value1.f1 + value2.f1);
        }
    });

    out.print("reduce");
    environment.execute("Reduce Demo");
}

    Union

    DataStream* → DataStream

    两个或以上的流合并成一个，这两个流的数据类型必须一样

    对两个或者两个以上的DataStream进行union操作，产生一个包含所有DataStream元素的新DataStream.

    注意:如果你将一个DataStream跟它自己做union操作，在新的DataStream中，你将看到每一个元素都出现两次

    示例：

    private void testUnion() throws Exception {

    StreamExecutionEnvironment environment = StreamExecutionEnvironment.getExecutionEnvironment();
    environment.setParallelism(1);

    //DataSource类随机发送数据作为数据源，将两个流做union输出

    DataStream<Tuple2<String, Integer>> orangeStream = environment.addSource(new DataSource("orangeStream"));
    DataStream<Tuple2<String, Integer>> greenStream = environment.addSource(new DataSource("greenStream"));

    orangeStream.union(greenStream).print("union");
    environment.execute("Union Demo");
}

public class DataSource extends RichParallelSourceFunction<Tuple2<String, Integer>> {
    private static final Logger LOGGER = LoggerFactory.getLogger(DataSource.class);
    private volatile boolean running = true;
    private volatile String name;

    public DataSource(String name) {
        this.name = name;
    }

    @Override
    public void run(SourceContext<Tuple2<String, Integer>> ctx) throws Exception {

        Random random = new Random();
        int bound = 100;
        final long numElements = 6;
        int i = 0;

        while (running && i < numElements) {
            Thread.sleep(1500);
            Tuple2 data = new Tuple2<>("foo", random.nextInt(bound));
            ctx.collect(data);
            LOGGER.info(Thread.currentThread().getId() + "-" + this.name + "-sand data:" + data);
            i++;
        }
    }

    @Override
    public void cancel() {
        running = false;
    }
}

    Connect & CoFlatMap

    Connect：DataStream,DataStream → ConnectedStreams

    CoFlatMap：ConnectedStreams → DataStream

    Connect可以连接两个不同数据类型的流，这个是和union最主要的区别；其次union支持2个以上流的合并，而Connect只支持2个流

    可以借助CoFlatMap将不同类型的流进行类型统一等操作。

    示例：

    private void testConnect() throws Exception {

    StreamExecutionEnvironment environment = StreamExecutionEnvironment.getExecutionEnvironment();
    environment.setParallelism(1);

    //DataSource1模拟流的数据类型为Tuple2，DataSource2模拟流的数据类型为Tuple3

    DataStream<Tuple2<String, Integer>> orangeStream = environment.addSource(new ConnectDataSource1());
    DataStream<Tuple3<String, Integer, Integer>> greenStream = environment.addSource(new ConnectDataSource2());

    //通过 Connect 连接两个数据流

    //通过 CoFlatMap 将数据做一个类型格式统一

    orangeStream.connect(greenStream).flatMap(new CoFlatMapFunction<Tuple2<String, Integer>, Tuple3<String, Integer, Integer>, Object>() {
        @Override
        public void flatMap1(Tuple2<String, Integer> value, Collector<Object> out) {
            if (!value.f0.contains("@")) {
                out.collect(new Tuple3<>(value.f0, value.f1, RandomUtils.nextInt(0, value.f1)));
            }
        }
        @Override
        public void flatMap2(Tuple3<String, Integer, Integer> value, Collector<Object> out) {
            for (String s : value.f0.split("@")) {
                out.collect(new Tuple3<>(s, value.f1, value.f2));
            }
        }
    }).print("Connect");

    environment.execute("Connect Demo");
}

public class ConnectDataSource1 extends RichParallelSourceFunction<Tuple2<String, Integer>> {

    private static final Logger LOGGER = LoggerFactory.getLogger(ConnectDataSource1.class);

    private volatile boolean running = true;

    @Override
    public void run(SourceContext<Tuple2<String, Integer>> ctx) throws Exception {

        int bound = 50;
        String[] keys = new String[]{"foo@xxyz", "bar", "baz"};

        final long numElements = RandomUtils.nextLong(10, 20);
        int i = 0;

        while (running && i < numElements) {
            Thread.sleep(RandomUtils.nextLong(1, 5) * 1000L);
            Tuple2 data = new Tuple2<>(keys[RandomUtils.nextInt(0, 3)], RandomUtils.nextInt(0, bound));
            ctx.collect(data);
            LOGGER.info(Thread.currentThread().getId() + "-sand data:" + data);
            i++;
        }
    }

    @Override
    public void cancel() {
        running = false;
    }
}

    private static final Logger LOGGER = LoggerFactory.getLogger(ConnectDataSource2.class);

    Iterate

    DataStream → IterativeStream → DataStream

    Iterate提供了一种流计算里面的类似递归方法

    在流中创建”反馈（feedback）”循环，通过将一个算子的输出重定向到某个先前的算子。这对于定义不断更新模型的算法特别有用。

    示例：

    private void testIterate() throws Exception {

    StreamExecutionEnvironment environment = StreamExecutionEnvironment.getExecutionEnvironment();
    environment.setParallelism(1);


    //以流开始并连续迭代
    //大于25的元素将被发送回反馈（feedback）通道，继续迭代，其余元素将向下游转发，离开迭代。


    //创建数据源
    DataStream<Tuple2<String, Integer>> orangeStream = environment.addSource(new IterateDataSource());

    IterativeStream<Tuple2<String, Integer>> iteration = orangeStream.iterate(5000);

    //每次对数字 减五
    SingleOutputStreamOperator<Tuple2<String, Integer>> iterationBody = iteration.map(new MapFunction<Tuple2<String, Integer>, Tuple2<String, Integer>>() {
        @Override
        public Tuple2<String, Integer> map(Tuple2<String, Integer> value) {
            return new Tuple2<>(value.f0, value.f1 - 5);
        }
    });

    //迭代依据（如果大于25，则循环迭代）
    SingleOutputStreamOperator<Tuple2<String, Integer>> feedback = iterationBody.filter(new FilterFunction<Tuple2<String, Integer>>() {
        @Override
        public boolean filter(Tuple2<String, Integer> value) {
            return value.f1 > 25;
        }
    });

    //这里设置feedback这个数据流是被反馈的通道，只要是value>25的数据都会被重新迭代计算。
    iteration.closeWith(feedback);

    DataStream<Tuple2<String, Integer>> output = iterationBody.filter(new FilterFunction<Tuple2<String, Integer>>() {
        @Override
        public boolean filter(Tuple2<String, Integer> value) {
            return value.f1 <= 25;
        }
    });

    feedback.print("Iterate feedback");
    output.print("Iterate output");
    environment.execute("Iterate Demo");
}

public class IterateDataSource extends RichParallelSourceFunction<Tuple2<String, Integer>> {

    private static final Logger LOGGER = LoggerFactory.getLogger(IterateDataSource.class);

    private volatile boolean running = true;

    @Override
    public void run(SourceContext<Tuple2<String, Integer>> ctx) throws Exception {
        int bound = 20;
        String[] keys = new String[]{"foo", "bar", "baz"};

        final long numElements = RandomUtils.nextLong(10, 20);
        int i = 0;

        //为了更清晰，临时注释
        //while (running && i < numElements) {
            Thread.sleep(RandomUtils.nextLong(1, 5) * 1000L);
            if (i == 0) {
                ctx.collect(new Tuple2<>(keys[RandomUtils.nextInt(0, 3)], 36));
            } else {
                Tuple2 data = new Tuple2<>(keys[RandomUtils.nextInt(0, 3)], RandomUtils.nextInt(10, bound));
                ctx.collect(data);
                LOGGER.info(Thread.currentThread().getId() + "-sand data:" + data);
            }
            i++;
        //}
    }

    @Override
    public void cancel() {
        running = false;
    }
}

    Split

    DataStream → SplitStream

    根据某些特征把一个DataStream拆分成两个或者多个DataStream.

    Select

    SplitStream → DataStream

    从一个SplitStream中获取一个或者多个DataStream

    Fold

    KeyedStream → DataStream

    合并当前元素和前一次折叠操作的结果，并产生一个新的值

    下面的fold函数就是当我们输入一个 (1,2,3,4,5)的序列, 将会产生一下面的句子:”start-1”, “start-1-2”, “start-1-2-3”,

    类似追加的效果

    Window

    KeyedStream → WindowedStream

    Windows 是在一个分区的 KeyedStreams中定义的. Windows 根据某些特性将每个key的数据进行分组 (例如:在5秒内到达的数据)

    WindowAll

    DataStream → AllWindowedStream

    Windows 可以在一个常规的 DataStreams中定义. Windows 根据某些特性对所有的流

    Window Join

    DataStream,DataStream → DataStream

    根据一个给定的key和window对两个DataStream做join操作.
