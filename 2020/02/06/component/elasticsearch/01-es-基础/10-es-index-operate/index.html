    elasticsearch index(索引) 操作

    索引名格式限制

    • 仅小写

    • 不能包含\ , / , * , ？ , “ , < , > , | , ‘’(空格字符) , 、 , #

    • 7.0版本之前可以含有’:’冒号；7.0版本之后不再支持

    • 不允许 以 - , _ , + 开头

    • 索引名称长度不能超过 255字节

    手动创建索引

    创建一个空索引

    PUT twitter

curl -XPUT 'http://127.0.0.1:9200/twitter'

    返回结果

    {
  "acknowledged" : true,
  "shards_acknowledged" : true,
  "index" : "facebook"
}

    acknowledged：

    集群是否成功创建索引

    shards_acknowledged：

    索引中每个分片是否成功启动所需数量的分片副本

    手动创建索引 并 设置索引分片

    手动创建索引，并 设置索引的 分片数量 和 副本分片数量

    PUT twitter
{
  "settings": {
    "index": {
      "number_of_shards": 5,
      "number_of_replicas": 0
    }
  }
}

    OR

    PUT twitter
{
    "settings" : {
        "number_of_shards" : 5,
        "number_of_replicas" : 0
    }
}

    number_of_shards：

    每个索引的主分片数，默认值是5，索引创建后不能修改

    number_of_replicas：

    每个主分片的副本数，默认值是1，这个配置可以随时修改

    如果要实时修改 number_of_replicas ，使用如下命令

    PUT /twitter/_settings
{
    "number_of_replicas": 1
}

    手动创建索引 并 设置映射

    关于”映射”,请移步 05-elasticsearch-映射_Mapping

    创建名称’twitter’的索引；

    设置5个主分片，0个副本分片；

    mapping设置三个字段，并分别设置字段类型。

    PUT twitter
{
  "settings": {
    "number_of_shards": 5,
    "number_of_replicas": 0
  },
  "mappings": {
    "_doc": {
      "properties": {
        "user": {
          "type": "text"
        },
        "post_date": {
          "type": "keyword"
        },
        "message": {
          "type": "text"
        }
      }
    }
  }
}

    手动创建索引 并 设置映射(2)

    先创建一个索引，然后再追加mappings设置

    PUT twitter
{
  "settings": {
    "index": {
      "number_of_shards": 5,
      "number_of_replicas": 0
    }
  }
}

    追加mappings设置

    put twitter/_mapping/_doc
{
  "properties": {
    "user": {
      "type": "text"
    },
    "post_date": {
      "type": "keyword"
    },
    "message": {
      "type": "text"
    }
  }
}

    查询 索引 Mappings映射

    查询twitter索引的映射

    GET /twitter/_mapping/_doc

GET /_mapping/_doc

    查询 所有索引的 Mappings映射

    GET /_all/_mapping

GET /_mapping

    快速创建索引（自动创建索引）

    使用 ‘PUT’ 方法，将数据插入到索引中。

    如果索引还不存在，会自动创建一个索引（通过 action.auto_create_index 配置，默认为true）

    例子：

    创建名为’twitter’的索引

    PUT twitter/_doc/1
{
    "user" : "kimchy",
    "post_date" : "2019-10-30",
    "message" : "trying out Elasticsearch"
}

    返回结果

    {
  "_index" : "twitter",
  "_type" : "_doc",
  "_id" : "1",
  "_version" : 1,
  "result" : "created",
  "_shards" : {
    "total" : 2,
    "successful" : 2,
    "failed" : 0
  },
  "_seq_no" : 0,
  "_primary_term" : 1
}

    _shards 节点：

    索引操作相关信息

    total：

    索引操作执行多少个分片副本，包含主副本分片

针对这条数据，一条数据只会写在一个分片上；

也就是会写在它的一个主分片、一个副本分片，一共两个。


注意：total 要看 number_of_replicas、number_of_shards 的设置。

举个栗子：

    twitter索引有5个主分片，每个主分片0个副本分片。

    再执行上面的put操作，返回的 total 就是1

    successful：

    索引成功操作分片的数量

    (注意，自动创建索引，使用的是动态映射，生产环境慎用。)

    删除索引

    删除索引，使用 DELETE 关键字

    DELETE twitter

curl -XDELETE 'http://127.0.0.1:9200/twitter?pretty'

    查询索引相关信息

    GET twitter

curl -XGET '127.0.0.1:9200/twitter?pretty'

curl -XGET 'http://127.0.0.1:9200/twitter?pretty'

    索引是否存在

    查询索引是否存在

    HEAD twitter

    返回结果

    200 - OK

    打开/关闭 索引

    为了维护index索引的元数据，或是其他原因， index索引 允许 关闭/打开。

    索引关闭之后，会被禁止读写。

    关闭索引几乎没有集群消耗。

    关闭索引

    POST /twitter/_close

curl -XPOST 'http://127.0.0.1:9200/twitter/_close?pretty'

    打开索引

    post /twitter/_open

curl -XPOST 'http://127.0.0.1:9200/twitter/_open?pretty'

    类型是否存在

    检查twitter索引中是否存在_doc类型

    HEAD twitter/_mapping/_doc
