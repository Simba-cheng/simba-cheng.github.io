    Spring 支持两种类型的事务管理：

    1.编程式事务管理

    2.声明式事务管理

    Spring的事务管理器

    Spring 将 事务管理 委托给 底层具体的持久化实现框架完成。

    因此，Spring为不同的持久化框架提供了 PlatformTransactionManager 接口的实现类，如下

    事务 说明
    org.springframework.orm.jpa.JpaTransactionManager 使用JPA进行持久化时，使用该事务管理器
    org.springframework.orm.hibernate4.HibernateTransactionManager 使用 Hibernate3.0 版本进行持久化，使用该事务管理器
    org.springframework.jdbc.datasource.DataSourceTransactionManager 使用 Spring JDBC、iBatis 等基于DataSource数据源的持久化技术时，使用该事务管理器。
    org.springframework.transaction.jta.JtaTransactionManager 具有多个数据源的全局事务，使用该事务管理器（不管采用何种持久化技术）

    这些事务管理器 都是对 特定事务实现框架 的代理。

    这样，我们就可以通过 Spring 所提交的高级抽象 对不同种类的事务 实现使用相同的方式进行管理

    要实现事务管理，首先要在 Spring中配置好相应的事务管理器，为事务管理器指定数据资源以及一些其他事务管理控制属性。

    例如：

    使用JDBC，基于数据源的Connection访问数据库，所以使用 DataSourceTransactionManager 事务管理器。

    c3p0DataSource即一个数据源，将数据源配置到 transactionManager 事务管理器中。

    在幕后，DataSourceTransactionManager 使用DataSource的Connection的commit()、rollback()等方法管理事务。

    <!-- 配置c3p0连接池 -->
<bean id="c3p0DataSource" class="com.mchange.v2.c3p0.ComboPooledDataSource">
    <property name="driverClass" value="${jdbc.driverClass}"/>
    <property name="jdbcUrl" value="${jdbc.url}"/>
    <property name="user" value="${jdbc.username}"/>
    <property name="password" value="${jdbc.password}"/>

    <property name="testConnectionOnCheckout" value="${jdbc.c3p0.testConnectionOnCheckout}"/>
    <property name="testConnectionOnCheckin" value="${jdbc.c3p0.testConnectionOnCheckin}"/>
    <property name="idleConnectionTestPeriod" value="${jdbc.c3p0.idleConnectionTestPeriod}"/>
    <property name="initialPoolSize" value="${jdbc.c3p0.initialPoolSize}"/>
    <property name="minPoolSize" value="${jdbc.c3p0.minPoolSize}"/>
    <property name="maxPoolSize" value="${jdbc.c3p0.maxPoolSize}"/>
    <property name="maxIdleTime" value="${jdbc.c3p0.maxIdleTime}"/>
</bean>

<!--Spring Jdbc 事务管理器-->
<bean id="transactionManager" class="org.springframework.jdbc.datasource.DataSourceTransactionManager">
    <property name="dataSource" ref="c3p0DataSource"/>
</bean>

    1.Spring 编程式事务管理

    编程式事务管理，允许 在对源代码编程 的帮助下 管理事务。

    灵活性比较好，但是很难维护(业务量大功能复杂时，编程式事务会很痛苦)，一般很少使用。

    Spring 提供了 org.springframework.transaction.support.TransactionTemplate 模板类，以满足一些特殊场合的需要。

    TransactionTemplate 是线程安全的。所以可以在多个业务类中共享 TransactionTemplate 实例进行事务管理。

    TransactionTemplate 有两个主要的方法：

    1.设置事务管理器

    void setTransactionManager(PlatformTransactionManager transactionManager)

    2.在回调接口中定义 需要以事务方式组织的 数据访问逻辑

    <T> T execute(TransactionCallback<T> action)

TransactionCallback执行完成会返回结果信息。

如果操作不需要返回结果，可以使用 TransactionCallback 子接口:TransactionCallbackWithoutResult

    实例

    applicationContext.xml 中定义 数据源、事务管理器 和 事务模版

    <!-- 配置c3p0连接池 -->
<bean id="c3p0DataSource" class="com.mchange.v2.c3p0.ComboPooledDataSource">
    <property name="driverClass" value="${jdbc.driverClass}"/>
    <property name="jdbcUrl" value="${jdbc.url}"/>
    <property name="user" value="${jdbc.username}"/>
    <property name="password" value="${jdbc.password}"/>

    <property name="testConnectionOnCheckout" value="${jdbc.c3p0.testConnectionOnCheckout}"/>
    <property name="testConnectionOnCheckin" value="${jdbc.c3p0.testConnectionOnCheckin}"/>
    <property name="idleConnectionTestPeriod" value="${jdbc.c3p0.idleConnectionTestPeriod}"/>
    <property name="initialPoolSize" value="${jdbc.c3p0.initialPoolSize}"/>
    <property name="minPoolSize" value="${jdbc.c3p0.minPoolSize}"/>
    <property name="maxPoolSize" value="${jdbc.c3p0.maxPoolSize}"/>
    <property name="maxIdleTime" value="${jdbc.c3p0.maxIdleTime}"/>

<!--Spring 事务管理器-->

<!--Spring 事务模版-->
<bean id="transactionTemplate" class="org.springframework.transaction.support.TransactionTemplate">

    Dao层

    @Component("accountDaoImpl")
public class AccountDaoImpl extends JdbcDaoSupport implements AccountDao {

    private static final Logger LOGGER = LoggerFactory.getLogger(AccountDaoImpl.class);

    @Autowired
    private javax.sql.DataSource c3p0DataSource;

    @PostConstruct
    private void initializeDataSource() {
        setDataSource(c3p0DataSource);
    }

    @Override
    public void outMoney(String out, Double money) {
        String sql = "update account set money = money - ? where name = ?";
        this.getJdbcTemplate().update(sql, money, out);
    }

    @Override
    public void inMoney(String in, Double money) {
        String sql = "update account set money = money + ? where name = ?";
        this.getJdbcTemplate().update(sql, money, in);
    }
}

    Service层

    @Service("accountServiceImpl")
public class AccountServiceImpl implements AccountService {

    private static final Logger LOGGER = LoggerFactory.getLogger(AccountServiceImpl.class);

    @Resource
    private AccountDao accountDaoImpl;

    @Autowired
    private TransactionTemplate transactionTemplate;

    //编程式事务管理
    @Override
    public void transfer(String out, String in, Double money) {

        transactionTemplate.execute(new TransactionCallbackWithoutResult() {
            @Override
            protected void doInTransactionWithoutResult(TransactionStatus transactionStatus) {
                //出账
                accountDaoImpl.outMoney(out, money);

                //制造异常信息
                //int i = 1 / 0;

                //入账
                accountDaoImpl.inMoney(in, money);
            }
        });
    }
}

    2.Spring XML 声明式事务管理

    使用声明式事务管理，这种方式对代码的侵入性最小，可以让事务管理代码完全从业务代码中移除，符合 非侵入式轻量级容器的理念。

    Spring的声明式事务管理是通过Spring AOP实现的，通过事务的声明性信息，Spring负责将 事务管理增强逻辑 动态编织 到 业务方法相应的连接点中。

    这些逻辑包含 获取线程绑定资源、开始事务、提交/回滚事务、进行异常转换和处理 等工作。

    Spring提供 TransactionProxyFactoryBean 代理类，对需要事务管理的业务类进行代理，以便实施事务功能的增强。

    例如：

    1.配置 dataSource，将其引入到DataSourceTransactionManager 事务管理器中。

    2.然后配置 TransactionProxyFactoryBean 代理工厂类，将 事务管理器 和 需要事务增强的servicebean引入其中。

    3.最后在注入service时，使用增强类即可

    <!--Spring Jdbc 事务管理器-->
<bean id="transactionManager" class="org.springframework.jdbc.datasource.DataSourceTransactionManager">
    <property name="dataSource" ref="c3p0DataSource"/>
</bean>

<!--配置增强处理类-->
<bean id="accountServiceProxy" class="org.springframework.transaction.interceptor.TransactionProxyFactoryBean">
    <!-- 配置目标对象 -->
    <property name="target" ref="accountServiceImpl"/>
    <!-- 注入事务管理器 -->
    <property name="transactionManager" ref="transactionManager"/>
    <!-- 注入事务的属性 -->
    <property name="transactionAttributes">
        <props>
            <prop key="transfer">PROPAGATION_REQUIRED</prop>
        </props>
    </property>
</bean>

    通过 TransactionProxyFactoryBean 对业务类进行代理，织入事务增强功能。

    通过’target’属性指定需要代理的目标bean。

    通过键值对配置业务方法的事务属性信息，key可以使用通配符，例如’get‘代表类中的所有以’get’为前缀的方法(如果key=’‘，则匹配类中所有的方法)。

    注意：

    如果 TransactionProxyFactoryBean 的optimize属性为true。

即<property name="optimize" value="true"/>

Spring会自动通过CGLib动态代理为 需要代理的目标bean 生成的子类进行代理。

这时，key="*" 不但匹配所有目标bean接口的方法，也匹配 目标Bean子类中的方法。

    异常回滚/提交规则

    节点中的值为事务属性信息，格式如下

    传播行为 隔离级别(可选) 是否为只读事务(可选) 发生这些异常时回滚事务(可选) 发生这些异常时照样提交事务(可选)
    PROPAGATION_REQUIRED IOSLATION readOnly _Exceptions +Exceptions

    例如：

    <prop key=“select*”>PROPAGATION_REQUIRED,readOnly</prop>

<prop key="save*">PROPAGATION_REQUIRED,-ApplicationException,+BusinessException</prop>

    传播行为是唯一必须提供的配置项，不过的值为空时，也不会发生配置异常，但对应的匹配方法不会应用事务。

    传播行为

    当事务方法被另一个事务方法调用时，必须指定事务应该如何传播。

    传播行为 含义
    PROPAGATION_REQUIRED 当前方法必须运行在事务中。
    如果当前事务存在，方法将会在该事务中运行。
    否则，会启动一个新的事务
    PROPAGATION_SUPPORTS 当前方法不需要事务上下文，
    但是如果存在当前事务的话，
    那么该方法会在这个事务中运行
    PROPAGATION_MANDATORY 该方法必须在事务中运行，如果当前事务不存在，则会抛出一个异常
    PROPAGATION_REQUIRED_NEW 当前方法必须运行在它自己的事务中。一个新的事务将被启动。
    如果存在当前事务，在该方法执行期间，当前事务会被挂起。
    如果使用JTATransactionManager的话，则需要访问TransactionManager
    PROPAGATION_NOT_SUPPORTED 该方法不应该运行在事务中。
    如果存在当前事务，在该方法运行期间，当前事务将被挂起。
    如果使用JTATransactionManager的话，则需要访问TransactionManager
    PROPAGATION_NEVER 当前方法不应该运行在事务上下文中。
    如果当前正有一个事务在运行，则会抛出异常
    PROPAGATION_NESTED 如果当前已经存在一个事务，那么该方法将会在嵌套事务中运行。
    嵌套的事务可以独立于当前事务进行单独地提交或回滚。
    如果当前事务不存在，那么其行为与PROPAGATION_REQUIRED一样。
    注意各厂商对这种传播行为的支持是有所差异的。
    可以参考资源管理器的文档来确认它们是否支持嵌套事务

    3.Spring 注解 声明式事务管理

    声明式事务，除了XML的事务配置之外，Sping还提供了基于注解的事务配置。

    通过 @Transactional 对需要事务增强的Bean接口、实现类、方法进行标记。

    在容器中配置基于注解的事务增强驱动，即可启动 注解声明式事务。

    <!--Spring Jdbc 事务管理器-->
<bean id="transactionManager" class="org.springframework.jdbc.datasource.DataSourceTransactionManager">
    <property name="dataSource" ref="c3p0DataSource"/>
</bean>

<!--开启注解事务-->
<tx:annotation-driven transaction-manager="transactionManager"/>

    @Transactional 注解的配置和基于xml的配置方式一样，它有一套默认的事务属性。一般情况下，使用默认属性即可

    事务传播行为：PROPAGATION_REQUIRED
事务隔离级别：ISOLATION_DEFAULT
读写事务属性：读/写事务
超时时间：依赖底层事务系统的默认值
回滚设置：任何运行期异常引发回滚，检查型异常不会引发回滚

    当然，如果默认的设置不适合，可以进行手工配置。


    @Transactional 注解可以被应用于接口、接口方法、类、类public方法上。

    但Spring建议，在业务实现类上使用 @Transactional 注解。

    当然也可以在业务接口上使用 @Transactional注解，不过当业务复杂，可能会有隐患。

    因为注解不能被继承，所以业务接口上标注的@Transactional注解不会被实现的业务类继承。

    通过如下配置，可以启动子类代理：

    <tx:annotation-driven transaction-manager="transactionManager" proxy-target-class="true"/>

    4.AspectJ 切面事务管理

    Spring还提供了AspectJ切面的支持。

    为 @Transactional 注解的业务类提供事务增强。

    实例

    applicationContext.xml

    <!-- 配置c3p0连接池 -->
<bean id="c3p0DataSource" class="com.mchange.v2.c3p0.ComboPooledDataSource">
    <property name="driverClass" value="${jdbc.driverClass}"/>
    <property name="jdbcUrl" value="${jdbc.url}"/>
    <property name="user" value="${jdbc.username}"/>
    <property name="password" value="${jdbc.password}"/>

    <property name="testConnectionOnCheckout" value="${jdbc.c3p0.testConnectionOnCheckout}"/>
    <property name="testConnectionOnCheckin" value="${jdbc.c3p0.testConnectionOnCheckin}"/>
    <property name="idleConnectionTestPeriod" value="${jdbc.c3p0.idleConnectionTestPeriod}"/>
    <property name="initialPoolSize" value="${jdbc.c3p0.initialPoolSize}"/>
    <property name="minPoolSize" value="${jdbc.c3p0.minPoolSize}"/>
    <property name="maxPoolSize" value="${jdbc.c3p0.maxPoolSize}"/>
    <property name="maxIdleTime" value="${jdbc.c3p0.maxIdleTime}"/>
</bean>

<!--Spring Jdbc 事务管理器-->
<bean id="transactionManager" class="org.springframework.jdbc.datasource.DataSourceTransactionManager">
    <property name="dataSource" ref="c3p0DataSource"/>
</bean>

<!-- 配置事务的增强 -->
<tx:advice id="txAdvice" transaction-manager="transactionManager">
    <tx:attributes>
        <tx:method name="transfer" propagation="REQUIRED"/>
    </tx:attributes>
</tx:advice>

<!-- 配置切面 -->
<aop:config>
    <!-- 配置切点 -->
    <aop:pointcut expression="execution(* com.transaction.service.AccountServiceFour*.*(..))" id="pointcut1"/>
    <!-- 配置切面 -->
    <aop:advisor advice-ref="txAdvice" pointcut-ref="pointcut1"/>
</aop:config>

    Dao层

    @Component("accountDaoImpl")
public class AccountDaoImpl extends JdbcDaoSupport implements AccountDao {

    private static final Logger LOGGER = LoggerFactory.getLogger(AccountDaoImpl.class);

    @Autowired
    private javax.sql.DataSource c3p0DataSource;

    @PostConstruct
    private void initializeDataSource() {
        setDataSource(c3p0DataSource);
    }

    @Override
    public void outMoney(String out, Double money) {
        String sql = "update account set money = money - ? where name = ?";
        this.getJdbcTemplate().update(sql, money, out);
    }

    @Override
    public void inMoney(String in, Double money) {
        String sql = "update account set money = money + ? where name = ?";
        this.getJdbcTemplate().update(sql, money, in);
    }
}
