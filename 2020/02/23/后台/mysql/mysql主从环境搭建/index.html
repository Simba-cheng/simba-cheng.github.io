    mysql主从环境搭建

    业务背景

    部门业务需求扩张，承接海量商品商品数据，需自己组建一个商品库。

    在方案设计时，选定用MySQL在存储商品数据。

    技术细节中，设定MySQL采用主从同步的方式，系统A写主库，系统B读从库(备库)，采用读写分离的方式。

    技术背景

    MySQL主从同步 解决的基本问题是：让一台服务器的数据与其他服务器保持同步。

    一台主库的数据可以同步到多台备库上，备库本身也可以配置成为另一台的服务器的主库。

    MySQL支持两种复制方式：基于行的复制、基于语句的复制。

    通过复制同步方式，可以将读操作指向备库来获得更好的读扩展。

    注意：在同一时间点备库上的数据可能与主库存在不一致，并且无法保证主备之间的延迟。

    mysql主从环境搭建

    在搭建主从环境之前，需要在各个节点安装好单机的MySQL，然后再通过主从配置，达到主从同步的目的。

    注意：各节点MySQL版本、安装路径 等信息，建议都保持一致。

    单机、主从安装可以参考这篇( linux 安装mysql8.0.x )，这里就不赘述

    主从角色配置

    IP 角色 主从配置
    172.*.*.130 master # 节点唯一id值
    server-id=1
    # 开启二进制日志
    log-bin=mysql-bin
    # 指定日志格式 有mixed、row、statement 推荐mixed
    binlog-format=mixed
    172.*.*.131 slave # 节点唯一id值
    server-id=2
    # 开启二进制日志
    log-bin=mysql-bin
    172.*.*.132 slave # 节点唯一id值
    server-id=3
    # 开启二进制日志
    log-bin=mysql-bin

    相关命令

    查看Master的状态

    show master status;

    启动Slave(在slave节点执行)

    start slave;

    查看 slave 的状态(slave节点执行)

    show slave status\G

    验证

    当下面两个状态都为yes，说明主从同步正常

    Slave_IO_Running: Yes
    Slave_SQL_Running: Yes

也可以在130的mysql数据库中，创建一张表，写入一些数据，然后看131、132 从数据库有没有复制同步
