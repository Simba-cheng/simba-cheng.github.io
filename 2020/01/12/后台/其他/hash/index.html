    老早之前看 HashMap 源码时，里面有hash；

    之前看 布隆过滤器/BitMap 其中也有hash；

    于是来了兴趣了，什么是hash？

    什么是hash

    较为抽象的解释

    hash(hash函数) 相当于将原空间的数据，映射到另一个空间。

在 关键字 与 记录的存储地址 之间建立的一种对应关系叫 哈希函数。
哈希函数就是一种映射，是从关键字到存储地址的映射。

    直观解释起来

    将任意长度的'数据值m'，通过hash算法转换成固定长度的'输出值h'，这个'输出值h'，就是hash值
或
将任意长度的原始数据压缩为一个固定长度的数字指纹

    一个好的hash函数应该具备以下三点：

    1.抗碰撞性 (尽量避免碰撞)
2.抗篡改性
3.查找效率

    由于场景不同，hash函数在 数据结构 和 密码学 中的用途并不相同，所以各自 算法设计/实现 的侧重点也不一样。

    (篇幅有限，只了解下hash碰撞….)

    hash碰撞问题

    两个不同的输入值，从同一个hash函数计算出相同的hash值的现象叫hash碰撞。

    同一hash函数计算的hash值不同，入参值肯定不同；
    同一hash函数计算的hash值相同，入参值可能不相同(hash碰撞)

    现在对Java中equals和hashCode有了更深刻的理解

    'equals相等，hashCode一定相同；hashCode相同，equals不一定相同'

    hash碰撞的问题是无法避免的。

    既然无法避免，只能尽可能减少碰撞带来的损失；

    hash碰撞处理方法

    hash碰撞的处理方法，这个应该是:基于 hash算法/函数 实现的数据结构自己的实现处理

    (hash函数/算法 本身并不能避免碰撞/冲突)

    开放寻址法

    数据结构：散列表（hash table）

    插入一个新数据，出现了hash碰撞/冲突，重新探查一个位置，直至找到一个空位置存储

    开发寻址法有三种实现：线性探查、二次探查 和 双重散列

    链接法

    数据结构：HashMap

    插入一个新数据，出现了hash碰撞/冲突，把同一位置的所有元素，放入一个链表中

    同 ‘开放寻址法’不同，链接法 会采用额外的数据结构来处理碰撞。

    String.hashCode

    看一个例子，String.hashCode方法

    public int hashCode() {
    int h = hash;
    if (h == 0 && value.length > 0) {
        char val[] = value;

        for (int i = 0; i < value.length; i++) {
            h = 31 * h + val[i];
        }
        hash = h;
    }
    return h;
}

    很简洁的乘加迭代运算

    运行下面这段代码，计算’a’的hashCode，输出结果是97

    String str = "a";
int hashCode = str.hashCode();
System.out.println(hashCode);

    通过debug可以看出，’a’字母，在char数组中显示的十进制数字是97，转换成二进制是1100001，不足8位，前面补零，也就是01100001

    二进制转换成十六进制就是61，然后对照ASCII码表，正好对应的就是’a’

    十进制 二进制 十六进制 字符
    97 01100001 61 a

    一个字母的时候，没有循环计算，再加一个字母，运行下面这个代码，debug看下

    String str = "aa";
int hashCode = str.hashCode();
System.out.println(hashCode);

    第一个循环结果是97，没问题，第二次循环，进行乘加迭代
    31 * 97 + 97 = 3104

    HashMap

    看下HashMap中hash算法，可以摘出来，单独放在main()方法中运行

    static final int hash(Object key) {
    int h;
    // h = key.hashCode() 为第一步 取hashCode值
    // h ^ (h >>> 16)  为第二步 高位参与运算
    return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}

    核心的是这一步

    这里的Hash算法本质上就是三步：取key的hashCode值、高位运算、取模运算

    (h = key.hashCode()) ^ (h >>> 16)

    ‘key的hashCode值’ 和 ‘hashCode值的无符号右移’ 位异或运算(两个数转为二进制，然后从高位开始比较，如果相同则为0，不相同则为1)

    h = key.hashCode = 97，转换成二进制是(0110 0001)
    h >>> 16 = 0，h的二进制不满16位，用0补足，最后就是0了

    0110 0001
    ^
    0000 0000
    =
    0110 0001

    0110 0001，值和原来一样，转换成十进制，就是97
