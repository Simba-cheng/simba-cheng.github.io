  1. 在电脑 A 上挂载电脑 B 的 SMB 共享

    1. 确认 SMB 共享配置

      • 在电脑 B 上，确保 SMB 共享已启用，且硬盘柜的路径（如 /Users/{your.dir}/Pictures/photos 和 /Users/{your.dir}/photoprism/storage）已共享。
      • 记下电脑 B 的 IP 地址（如 192.168.1.100）和共享路径。
      • 确保电脑 A 和 B 在同一局域网内，且防火墙允许 SMB 协议（默认端口 445）。

    2. 在电脑 A 上挂载 SMB 共享

      • 通过 Finder 挂载

        • 打开 Finder，按 Command + K（或菜单栏选择“前往” -> “连接到服务器”）。
        • 输入 SMB 共享地址，例如：smb://192.168.1.100/Pictures/photos。
        • 输入电脑 B 的用户名和密码（如需要）。
        • 挂载后，SMB 共享会出现在 /Volumes 下，例如 /Volumes/photos。

      • 通过命令行挂载（更灵活）：

        • 替换 username 和 password 为电脑 B 的 SMB 共享凭据。

        • 替换 192.168.1.100 为电脑 B 的实际 IP 地址。

        • 挂载后，检查 /Volumes/photos 和 /Volumes/photoprism_storage 是否包含预期文件：

    3. 验证挂载稳定性

      • 确保挂载的 SMB 共享可读写

      • 如果挂载失败，检查

        • 网络连接街连接（电脑 A 和 B 间的网络是否稳定）。
        • SMB 共享的权限设置（确保电脑 A 的用户有读写权限）。
        • macOS 的 SMB 挂载配置是否正确。

    • 将 /Users/{your.dir}/photoprism/storage 替换为 /Volumes/photoprism_storage。
    • 数据库文件存储路径（/Users/{your.dir}/mariadb/database）保留为本地路径，避免在 SMB 共享上存储数据库文件（可能导致性能或可靠性问题）。
