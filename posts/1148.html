1. 什么是零拷贝？

零拷贝的核心目标是 减少 CPU 复制数据的开销，提高 I/O 性能。在传统的文件传输 (如 read → write) 过程中，数据在 内核缓冲区 和 用户缓冲区 之间来回拷贝，导致不必要的 CPU 消耗和内存带宽占用。而 Linux 提供了 零拷贝技术，直接让数据在内核空间流转，而不经过用户空间，从而减少拷贝次数，提高传输效率。


2. 传统的文件拷贝操作

以下是传统的文件拷贝操作代码：

  • 详细数据流向：
    1. DMA拷贝：磁盘文件 -> 内核缓冲区（PageCache）
    2. CPU拷贝：内核缓冲区 -> 用户空间缓冲区（byte[4096]）
    3. CPU拷贝：用户空间缓冲区 -> socket缓冲区
    4. DMA拷贝：socket缓冲区 -> 目标磁盘文件
  • 上下文切换过程：
    1. read() 系统调用：用户态 -> 内核态
    2. read() 返回：内核态 -> 用户态
    3. write() 系统调用：用户态 -> 内核态
    4. write() 返回：内核态 -> 用户态

这里经历了 4 次数据拷贝和 4 次上下文切换，性能较差。


3. Java NIO 中的零拷贝实现

3.1. FileChannel.transferTo 方法

Java NIO (java.nio.channels.FileChannel#transferTo) 可以调用 Linux sendfile 系统调用，从而实现零拷贝。

transferTo 方法底层会直接调用 Linux 的 sendfile() 系统调用，避免数据进入 用户缓冲区，数据传输期间：

  • 只需要从磁盘复制到内核缓冲区
  • 再从内核缓冲区复制到 NIC 缓冲区
  • 只经历 2 次上下文切换

完全绕过用户空间，适用于不需要处理数据的纯拷贝场景。


3.2. 使用 MappedByteBuffer

这种方式使用了内存映射，将文件直接映射到内核缓冲区，同样可以避免用户空间和内核空间的数据拷贝。

适用场景：需要随机访问文件内容、需要修改部分文件内容。
